# Technical Decisions

## SearchPlatform 인터페이스를 만든 이유

플랫폼마다 검색 방식, 응답 구조, API가 모두 다르다.

공통 인터페이스를 사용하여
확장 가능한 구조를 선택하였다.

---

## Musinsa를 보류한 이유

API가 존재하지만

브라우저 환경 의존성이 높았다.

유지보수 비용을 고려하여
우선순위를 낮추었다.

---

## KREAM을 조사한 이유

HTML에 SSR 데이터가 포함되어 있었다.

공식 API가 없어도
검색 구현 가능성이 있다고 판단하였다.

---

## 공식 API 우선 전략

역공학보다

안정적인 API를 먼저 연동하여

프로젝트 완성도를 높이기로 결정하였다.
